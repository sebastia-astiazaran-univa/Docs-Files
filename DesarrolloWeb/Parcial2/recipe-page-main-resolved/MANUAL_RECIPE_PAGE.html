Manual para el proyecto Recipe Page

Para alumnos de preparatoria — Desarrollo Web

Este manual te guía paso a paso para resolver el reto Recipe page de Frontend Mentor usando HTML y CSS, con diseño adaptable a móvil y escritorio.

1. Introducción al reto

1.1 ¿Qué vas a construir?

Vas a crear una página web de una receta de omelette. La página tendrá:

  • Una imagen principal de la receta
  • Título y descripción
  • Tiempo de preparación
  • Lista de ingredientes
  • Pasos (instrucciones) numerados
  • Tabla de información nutricional

El objetivo es que tu página se parezca lo más posible al diseño de referencia.

1.2 Archivos importantes del proyecto

Dentro de la carpeta recipe-page-main encontrarás:

Archivo o carpetaPara qué sirve
index.htmlArchivo principal de la página (aquí escribirás el contenido)
assets/images/image-omelette.jpegImagen del omelette
style-guide.mdColores, fuentes y tamaños que debes usar
design/mobile-design.jpgCómo debe verse en celular
design/desktop-design.jpgCómo debe verse en computadora

Recomendación: Ten abierta una ventana con el diseño (mobile o desktop) para ir comparando mientras programas.

1.3 Frontend Mentor y el style-guide

El reto viene de Frontend Mentor. El archivo style-guide.md indica los colores (HSL), las fuentes (Young Serif y Outfit) y el tamaño de letra base (16px). Úsalo como referencia al aplicar estilos.

2. Preparar el entorno de trabajo

2.1 Abrir el proyecto en VS Code

  1. Abre VS Code.
  2. Menú Archivo → Abrir carpeta.
  3. Selecciona la carpeta recipe-page-main.
  4. Haz clic en Abrir.

2.2 Instalar Live Server (solo la primera vez)

  1. Panel de Extensiones (ícono de cuadrados).
  2. Busca Live Server.
  3. Instala la extensión Live Server (Ritwick Dey).
  4. Reinicia VS Code si te lo pide.

2.3 Ver la página en el navegador

  1. Abre el archivo index.html.
  2. Clic derecho → Open with Live Server.
  3. Se abrirá el navegador. Cada vez que guardes (Ctrl+S / Cmd+S), la página se actualizará sola.

3. Analizar el diseño y el contenido

Abre las imágenes de design/. Haz una lista de las 7 partes: imagen, título y descripción, tiempo de preparación, ingredientes, instrucciones, nutrición, pie de página.

Relaciona con HTML: contenedor → main o article; títulos → h1, h2, h3; párrafos → p; listas → ul/li (ingredientes), ol/li (instrucciones); nutrición → table o div por fila.

4. Construir la estructura HTML paso a paso

Paso 1: Estructura base: DOCTYPE, html, head (meta charset, viewport, title), body.

Paso 2: Contenedor principal: <main class="recipe-card"> ... </main>.

Paso 3: Imagen: <img src="./assets/images/image-omelette.jpeg" alt="Omelette en un plato">.

Paso 4: h1 con el título y p con la descripción.

Paso 5: Sección tiempo de preparación: section con h2/h3 y lista ul/li.

Paso 6: Ingredientes: h2 + ul/li.

Paso 7: Instrucciones: h2 + ol/li.

Paso 8: Nutrición: h2, párrafo y tabla o filas con nombre + valor.

Paso 9: Bloque attribution al final con tu nombre en “Coded by”.

5. Conectar la hoja de estilos CSS

Crear styles.css en la raíz y en el head de index.html añadir:

<link rel="stylesheet" href="styles.css">

Comprobar con body { background-color: #f0f0f0; } en el CSS.

6. Estilos base y tipografía

Reset básico: * { box-sizing: border-box; }, body { margin: 0; padding: 0; }. Usar colores y fuentes del style-guide.md (Young Serif, Outfit desde Google Fonts). Estilo de .recipe-card: fondo blanco, border-radius, max-width, padding. Títulos con Young Serif, párrafos 16px, color Stone 600.

7. Maquetar las secciones (móvil primero)

Imagen: width: 100%, border-radius. Tiempo de preparación: caja con fondo Rose 50, padding, border-radius. Ingredientes e instrucciones: márgenes y padding en listas. Nutrición: filas con dos columnas (tabla o display: flex), border-bottom entre filas.

8. Hacer la página responsive (escritorio)

Al final de styles.css:

@media (min-width: 768px) {
  .recipe-card {
    max-width: 736px;
    padding: 40px;
  }
}

Comparar con design/desktop-design.jpg.

9. Revisión final

  • Todas las secciones presentes.
  • Texto legible y buen contraste.
  • Se ve bien en móvil y en escritorio.
  • “Coded by” con tu nombre.
  • Colores y fuentes según style-guide.

Ejemplo guiado (cómo pensar el proyecto)

Parte 1: Observar el diseño y anotar: imagen → img, título → h1, cada bloque → section con h2 y listas o tabla.

Parte 2: Escribir primero la estructura (main + sections con títulos), luego rellenar cada sección.

Parte 3: Probar estilos: cambiar color de fondo, añadir borde a la tarjeta, modificar padding.

Parte 4: Añadir la media query y redimensionar la ventana para ver móvil vs escritorio.

¡Listo! Sigue el manual paso a paso y compara siempre con el diseño y el style-guide.md.
