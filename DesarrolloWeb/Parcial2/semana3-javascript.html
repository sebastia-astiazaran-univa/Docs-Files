Clase 9

Variables & Tipos de datos

JavaScript trabaja con distintos tipos de valores. Cada uno tiene un comportamiento diferente en la interfaz.

String

"FAQ"

Texto. Se usa para títulos, mensajes, estados descriptivos.

Number

42

Número. Útil en contadores, posiciones, animaciones.

Boolean

false

Verdadero o falso. Controla si algo está abierto o cerrado.

Function

toggleState()

Acción reutilizable. Recibe un valor y devuelve su opuesto.

Clase 10

DOM & Acordeón

El DOM es la representación del HTML como objetos. Con JavaScript podemos seleccionar elementos y escuchar eventos del usuario.

const button = document.querySelector('.faq-question');

button.addEventListener('click', () => {
  item.classList.toggle('is-open');
});

// CSS responde al estado:
.item.is-open .answer { display: block; }

FAQ Acordeón · Tarea 3

Cada botón usa aria-expanded y escucha el evento click.

Es el método del DOM que permite seleccionar el primer elemento que coincida con un selector CSS. Por ejemplo: document.querySelector('.btn') devuelve el primer elemento con clase btn. Para seleccionar todos, se usa querySelectorAll().

Agrega una clase si el elemento no la tiene, y la elimina si ya la tiene. Es el corazón de componentes tipo acordeón: item.classList.toggle('is-open') activa y desactiva el estado visual sin condiciones extra.

display:none elimina el elemento del flujo del documento: no ocupa espacio. visibility:hidden lo oculta visualmente pero mantiene su espacio. Para acordeones se prefiere controlar max-height y overflow:hidden para animar suavemente.

Los botones reales son accesibles por teclado, envían eventos de clic con Enter/Espacio y son reconocidos por lectores de pantalla. Un <div> necesita atributos y manejadores extra para lograr lo mismo; siempre es mejor usar el elemento semántico correcto.

Clase 11

Componentes interactivos

Además del acordeón, estos patrones aparecen en casi cualquier interfaz. No son obligatorios en la tarea, pero amplían el criterio de diseño.

Tabs

HTML semántico

La estructura define el significado. Usa <button>, <nav>, <section> en lugar de divs genéricos para que el navegador y los lectores de pantalla entiendan el contenido.

CSS como capa visual

Las clases de estado (como .is-open o .active) conectan JavaScript con los estilos. El CSS responde al DOM, no al revés.

JavaScript como comportamiento

JS escucha eventos y modifica clases o atributos. No debería contener lógica de presentación: eso es territorio del CSS. La separación de responsabilidades mantiene el código mantenible.

Clase 12

Accesibilidad básica

Una interacción que funciona para todos. Estos principios no son opcionales: son parte de diseñar bien.

🎯

Foco visible

Navegar con teclado debe tener indicador visual claro. El foco no se oculta, se diseña.

:focus-visible

🔲

Botones reales

Usa <button> para acciones. El navegador gestiona el teclado y el rol semántico automáticamente.

role="button"

🌗

Contraste

Texto sobre fondo: mínimo 4.5:1. Elementos grandes: 3:1. Afecta legibilidad en cualquier contexto.
